# Global Music Finder

[🇬🇧 English](README.md) | 🇯🇵 Japanese  

海外の音楽を検索しやすくする音楽検索アプリ。

言語別の楽曲検索、楽曲・アーティスト名のコピー、楽曲の試聴が可能。

HTML、CSS (SCSS)、JavaScript、iTunes APIを用いて開発。

## Background
韓国語や中国語の音楽を聴きたいと思ったとき、語学力がなく、検索文字を入力することに苦労した経験がありました。

知らない言語の音楽でも、もっと簡単に検索できるアプリがあれば便利だと思い、このアプリを制作しました。

## Features

### 言語別の楽曲検索
言語ごとに分かれたページから、目的の音楽を探せます。

### 楽曲・アーティスト名のコピー
曲名やアーティスト名をワンクリックでコピーできます。

知らない文字を入力することなく再検索できるため、アプリ内だけでなく外部サイトで検索するときにも活用できます。

### 楽曲の試聴
楽曲をその場で試聴できるため、別途音源を探す必要がなく、効率よく新しい音楽を見つけられます。

## Technology Choices
JavaScriptによるAPI連携の実装に重点を置きたかったため、HTML・SCSS・JavaScriptを採用しました。

APIには、無料かつ認証不要で利用できる iTunes Search API を選択しました。

また、多言語の楽曲を幅広く検索できることも選定理由の一つです。

## System Design

### Final Design
検索結果をスクロールしても言語選択タブが表示されたままになるよう、操作パネルを画面左側に固定しました。

また、言語ごとにアクセントカラーを統一することで、現在どの言語を選択しているかを視覚的に把握しやすいデザインを目指しました。

### Initial Design
当初は言語タブを画面上部に配置していました。

しかし、検索結果をスクロールすると操作パネルが画面外へ移動してしまい、言語を切り替えるたびにページ上部へ戻る必要がありました。

## Implementation Highlights
JavaScriptは機能単位でファイルや関数を分割し、保守性と拡張性を意識した構成にしました。

ページごとに必要なコードのみを読み込む構成とすることで、不要な処理を避けられるよう設計しています。

検索結果の生成にはテンプレートリテラルを採用しました。

`createElement()`を使って要素を一つずつ組み立てるのではなく、HTML構造をまとめて記述することで、複雑なDOMでも読みやすく保守しやすいコードを目指しました。

CSS設計にはBEMを採用しました。

コンポーネント単位でスタイルを管理しやすく、SCSSとも相性が良いためです。

また、言語ごとのテーマカラーはあらかじめSCSSファイルとして切り出し、SCSS変数とラジオボタンの状態を利用して切り替えることで、追加コードを最小限に抑えました。

## Challenges & Solutions
試聴ボタンの実装と状態管理に最も苦労しました。

当初は各検索結果に`<audio>`タグを配置していましたが、検索結果が増えるほど`<audio>`要素も増え、パフォーマンス上の課題が発生しました。

その後、別の実装方法も試しましたが、音源を正常に再生できませんでした。

最終的には、安定した再生を優先して`<audio>`タグによる実装を採用しています。

一方で、再生状態に応じたアイコンの切り替えは未実装のため、今後改善したい課題として残しています。

## Future Improvements
今後は、再生状態に応じて試聴ボタンのアイコンを切り替えられるよう改善したいと考えています。

現在は音源が再生されてもアイコンが変化しないため、再生中か停止中かを視覚的に判断しづらい状態です。

また、検索結果をスクロールしても検索ボックスを利用できるよう、画面上部に固定表示する予定です。

検索のたびにページ上部へ戻る必要がなくなり、より快適に利用できるよう改善したいと考えています。
